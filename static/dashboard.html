Manual de integración para desarrolladores

🔑 Autenticación: todas las llamadas requieren el header X-API-Key: yek_tu_key. El endpoint base es https://yepardtecore.cl/api

🚀 Inicio rápido

Integrar YeparDTEcore tiene 3 pasos: autenticarte, mandar el certificado del emisor y los datos del documento. Core firma, construye el XML y lo envía al SII por ti.

// Node.js / fetch const res = await fetch('https://yepardtecore.cl/api/firmar-y-enviar', { method: 'POST', headers: { 'X-API-Key': 'yek_tu_api_key', 'Content-Type': 'application/json' }, body: JSON.stringify({ emisor: { rut: '76123456-7', razon_social: 'Mi Empresa SpA', giro: 'Comercio', direccion: 'Av. Principal 123', comuna: 'Santiago', ciudad: 'Santiago', nro_resolucion: '0', fch_resolucion: '2000-01-01' }, pfx_base64: 'base64_del_certificado_p12', pfx_password: 'clave_del_certificado', caf_base64: 'base64_del_CAF_xml', tipo: 33, receptor: { rut: '12345678-9', nombre: 'Cliente SA' }, items: [{ nombre: 'Servicio', cantidad: 1, precio: 10000 }], ambiente: 'produccion' }) }); const data = await res.json(); console.log(data.track_id); // ID de seguimiento SII

📡 Endpoints disponibles

POST /api/firmar-y-enviar Firma y envía un DTE al SII (stateless) ▼

Endpoint principal. Recibe el certificado (.p12) y CAF del emisor en base64, construye el XML, lo firma y lo envía al SII. No guarda nada en base de datos.

CampoTipoRequeridoDescripción
emisorobjeto✓Datos del emisor (RUT, razón social, giro, dirección, resolución)
pfx_base64string✓Certificado .p12 del emisor en base64
pfx_passwordstring✓Contraseña del certificado
caf_base64string✓CAF XML del SII en base64
tipoint✓33=Factura, 34=F.Exenta, 39=Boleta, 52=Guía, 56=ND, 61=NC
receptorobjeto✓RUT y nombre del receptor
itemsarray✓Lista de productos/servicios con nombre, cantidad y precio
ambientestringopc"certificacion" o "produccion" (default: certificacion)
folio_actualintopcFolio a usar. Si no viene, usa el primero del CAF
auto_enviarboolopctrue = enviar al SII, false = solo firmar (default: true)
// Respuesta exitosa { "ok": true, "tipo": "Factura Electrónica", "folio": 195, "track_id": "0252066212", "estado": "RECIBIDO", "xml_firmado": "..." }
GET /api/estado/{track_id} Consulta estado de un envío en el SII ▼

Consulta al SII el estado de un envío usando su track_id. El SII no notifica proactivamente — debes consultar tú.

ParámetroTipoDescripción
track_idpathID de seguimiento devuelto por /firmar-y-enviar
rut_emisorqueryRUT del emisor del documento
tipoqueryTipo DTE (39/41 usa ventanilla de boletas)
ambientequery"certificacion" o "produccion"
// Ejemplo GET /api/estado/0252066212?rut_emisor=76123456-7&tipo=33&ambiente=produccion // Respuesta { "track_id": "0252066212", "estado": "ACEPTADO", "codigo_sii": 0, "glosa": "Envío Recibido OK", "aceptados": 1, "rechazados": 0 }
POST /api/generar-set Genera un set completo de certificación ▼

Genera el EnvioBOLETA/EnvioDTE completo con múltiples casos para certificación SII. Devuelve el sobre firmado listo para subir.

💡 Usado principalmente durante el proceso de certificación SII. Para emisión productiva usa /api/firmar-y-enviar.
GET /api/folios Consulta folios disponibles del emisor ▼
{ "folios": [ { "tipo": 33, "descripcion": "Factura", "desde": 1, "hasta": 100, "actual": 45, "disponibles": 56 } ] }
POST /api/generar-libro-desde-xml Genera libro de ventas, compras o guías ▼

Recibe los XML de EnvioDTE aprobados por el SII y genera el libro correspondiente firmado.

Manual de integración

Guía paso a paso para integrar YeparDTEcore en tu software

1

Obtén tu API Key

Tu API key se generó al registrarte y pagar la suscripción. La encuentras en la sección API Key de este panel. Es única para tu software y debes mantenerla secreta — nunca la expongas en el frontend o en código público.

🔐 Guárdala como variable de entorno en tu servidor:
DTECORE_API_KEY=yek_tu_key_aqui
2

Prepara los datos del emisor

Cada llamada a la API es stateless — debes enviar los datos del emisor en cada request. Esto significa que tu software guarda los datos de sus clientes (certificado, CAF, RUT) y los manda a Core en cada emisión. Core no los almacena.

Los datos que necesitas tener en tu BD por cada emisor de tu sistema:

DatoCómo obtenerlo
rut, razon_social, giroDatos del SII del emisor
direccion, comuna, ciudadDirección registrada en el SII
certificado_p12Archivo .p12 o .pfx del SII (firma electrónica)
certificado_passwordContraseña del certificado
caf_xml por tipoCAF descargado desde sii.cl → Mi SII → Folios
nro_resolucionNúmero de resolución del SII (generalmente "0")
fch_resolucionFecha de resolución (generalmente "2000-01-01")
3

Convierte el certificado y CAF a base64

La API recibe el certificado y el CAF como strings base64. Debes convertirlos antes de enviarlos. Ejemplo en Node.js y Python:

// Node.js const fs = require('fs'); const pfxBase64 = fs.readFileSync('certificado.p12').toString('base64'); const cafBase64 = fs.readFileSync('caf_33.xml').toString('base64');
# Python import base64 pfx_b64 = base64.b64encode(open('certificado.p12', 'rb').read()).decode() caf_b64 = base64.b64encode(open('caf_33.xml', 'rb').read()).decode()
4

Lleva el contador de folios

Core no guarda el contador de folios — eso lo maneja tu sistema. Debes guardar el próximo folio a usar por cada emisor y tipo de DTE, e incrementarlo después de cada emisión exitosa.

// Ejemplo de tabla en tu BD folios_emisor ( emisor_id INT, tipo_dte INT, -- 33, 39, 61, etc. folio_actual INT, -- próximo folio a usar folio_hasta INT, -- último folio del CAF actual caf_xml TEXT -- CAF en base64 ) // Flujo: 1. Leer folio_actual del emisor para tipo_dte 2. Enviar a Core con folio_actual 3. Si la respuesta es ok → incrementar folio_actual en tu BD 4. Si folio_actual > folio_hasta → solicitar nuevo CAF al SII
5

Emite tu primer DTE

Con todo listo, llama a POST /api/firmar-y-enviar. Aquí un ejemplo completo en Node.js:

// Emitir una factura electrónica (tipo 33) const resultado = await fetch('https://yepardtecore.cl/api/firmar-y-enviar', { method: 'POST', headers: { 'X-API-Key': process.env.DTECORE_API_KEY, 'Content-Type': 'application/json' }, body: JSON.stringify({ emisor: { rut: emisor.rut, razon_social: emisor.razon_social, giro: emisor.giro, direccion: emisor.direccion, comuna: emisor.comuna, ciudad: emisor.ciudad, nro_resolucion: emisor.nro_resolucion, fch_resolucion: emisor.fch_resolucion }, pfx_base64: emisor.certificado_p12_base64, pfx_password: emisor.certificado_password, caf_base64: emisor.caf_33_base64, folio_actual: emisor.folio_actual_33, tipo: 33, receptor: { rut: receptor.rut, nombre: receptor.razon_social, giro: receptor.giro, direccion: receptor.direccion, comuna: receptor.comuna, ciudad: receptor.ciudad }, items: [{ nombre: 'Servicio de consultoría', cantidad: 1, precio: 100000, exento: false }], ambiente: 'produccion', auto_enviar: true }) }); const data = await resultado.json(); if (data.ok) { // Guardar el track_id para consultar estado después await db.query('UPDATE folios SET folio_actual = folio_actual + 1 WHERE ...'); console.log('Track ID SII:', data.track_id); }
6

Consulta el estado en el SII

El SII no notifica — debes consultar el estado del envío usando el track_id que devolvió el paso anterior. Recomendamos consultar después de 30-60 segundos.

// Consultar estado const estado = await fetch( `https://yepardtecore.cl/api/estado/${trackId}?rut_emisor=${rut}&tipo=33&ambiente=produccion`, { headers: { 'X-API-Key': process.env.DTECORE_API_KEY } } ); const s = await estado.json(); // s.estado → "ACEPTADO" | "RECHAZADO" | "REPARO" | "PENDIENTE"
📋 Estados posibles:
ACEPTADO — El SII aceptó el documento correctamente.
RECHAZADO — El SII rechazó el documento. Revisa el campo glosa.
REPARO — Aceptado con observaciones menores.
PENDIENTE — El SII aún está procesando. Consulta en 5 minutos.

certificacion maullin.sii.cl Pruebas y certificación. Los documentos no tienen validez fiscal.
produccion palena.sii.cl Documentos reales con validez fiscal. Requiere resolución SII.
